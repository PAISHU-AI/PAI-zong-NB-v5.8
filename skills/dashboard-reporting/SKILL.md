# dashboard-reporting

## Purpose

Plan a dashboard or reporting page that supports repeated business review.

## Workflow

1. Identify audience, cadence, decisions, metrics, dimensions, data sources, and refresh needs.
2. Define KPI cards, chart sections, tables, filters, drilldowns, and alerts.
3. Specify metric definitions, denominators, time windows, and exclusions.
4. Arrange layout by decision priority.
5. Add data-quality, access, and validation requirements.

## Required Checks

- Do not invent live dashboard data or access.
- Do not hide metric definitions.
- Keep the dashboard focused on decisions, not decoration.

## Output

Return dashboard spec, KPI definitions, layout, chart list, filters, and validation checklist.
