# Seedance 2.0 — 创作内功：能力 · 运动轴 · 运镜语义 · 声音 · 冲突 · 迭代（中文）

> 按需加载模块。硬性上限与流程见 SKILL.md，@语法与运镜术语见 syntax.md。
> 这一份装的是"导演级判断"：选运动轴、定运镜的情绪动机、设计声音层、仲裁素材冲突、处理迭代修改、给抽象需求做默认补全。要做这些决定时打开它。

## 双入口模式（每次先认）

- **首尾帧模式**：只有 1–2 张图当首帧/尾帧 + 文字，不碰视频、音频、多素材参考、编辑、延长、卡点。直接传图配文。限制：智能多帧与主体参考不可用。
- **全能参考模式**：一切要 @引用 的任务——多图、视频、音频、运镜复刻、动作复刻、延长、编辑、融合、卡点、多角色对话、漫改、MV、纪录片、广告。
- **纯文字无素材**：默认全能参考，控制力更强。
- **判断原则**：需求一旦超过"一张图 + 一段文字"，走全能参考。

## 一条视频，一条运动主轴（三选一）

- **空间穿越**：镜头或主体物理位移——微距推到全景、街头跟到屋顶、舱内推向云层、穿过门进入另一空间。
- **状态蜕变**：主体形态/材质/色彩/身份/情绪转换——冰融化、金属锈蚀、花盛开、人物变身、城市从白昼转夜色。
- **情绪迁移**：整体氛围不可逆移动——温柔到失控、秩序到混乱、压抑到爆发、宁静到悬疑。

多个想法时，选视觉冲击最大、最能在 15 秒内闭环的一条，其余作变体或删去。一镜一意：禁止在 15 秒里塞进多个互相争抢的主题。

## 运镜的情绪语义（不只是术语，是动机）

术语表在 syntax.md；这里是每个运镜"为什么用"。运镜必须服务叙事，不堆术语。

- **推镜**：逼近真相、压迫、亲密、情绪升温。
- **拉镜**：揭示全貌、疏离、余韵、反转。
- **环绕**：审视、迷恋、困局、仪式感。
- **跟拍**：陪伴、追逐、纪实、代入。
- **低机位**：力量、英雄感、压迫。
- **高机位**：孤独、渺小、命运感。
- **甩镜 / 快速摇**：惊讶、转折、卡点、喜剧反差。
- **固定机位**：克制、冷观察、荒诞、纪录片真实。

## 光与材质（开拍前必答）

- **光是第一个演员**：说明光源、方向、色温、阴影、遮蔽或光影变化。没有光的描述，画面就是平的。
- **材质承载情绪**：金属/玻璃/水/火/皮肤/布料/尘埃/雾气/混凝土，都为气氛服务，不是背景板。
- **反平庸细节**：至少植入一个高熵细节——异常光斑、粒子扰动、材质裂变、反常运动、声画错位。让画面"记得住"。

## 声音是第二画面（五层 + 沉默）

声音是生成的，要么引导它，要么接受默认。**无音频素材也必须给出至少一层声音方案。**

- **环境音**：建立空间真实——风、雨、城市低频、室内电流声、脚步、衣料摩擦。
- **配乐**：控情绪曲线——写风格、节奏、起伏，是否参考 @音频/@视频。
- **人声**：直接写台词、语言/方言、语气、情绪；有音色参考写明来源（音色照 @视频X / @音频X）。
- **音效**：转场、撞击、出现、消失、卡点、戏剧性停顿。
- **沉默**：制造压迫、反转、情绪断裂——沉默是声音设计的一部分，不是缺失。

声音层多时写**优先级**，避免互相遮盖（如"人声压过配乐，环境音垫底"）。

## 画内文字协议（画面出现文字必写全五要素）

完整文字内容 + 出现时间 + 位置 + 动画方式 + 消失方式。

- 动画方式：渐显 / 从画面中央弹出 / 遮罩形式出现 / 弹出后逐渐淡化。
- 位置：视频中间 / 画面底部 / 主体背后 等空间描述。
- 多行按行写（"第一行…第二行…"）。短句优先；超过约 10 字的长句易变形，拆多行。
- 英文字体渲染通常比中文稳，中文复杂字形可能需多次尝试。

## 素材分配优先级（接近上限时砍弱留强）

混合输入上限 12 个文件，三类配额（图≤9/视频≤3/音频≤3）相加 15 但合并封顶 12，填不满全部。可靠分配：3–5 图 + 1–2 视频 + 1 音频。接近上限时按优先级取舍：

1. **锁身份**（最高）：人物脸、商品外观、品牌 Logo。
2. **锁动作/节奏**：参考视频、参考音频。
3. **锁氛围/风格**（最低）：场景图、色调图、海报图——接近上限先删这一类。

补：输入参考视频像素范围 640×640 至 834×1112；视频延长任务，生成时长**必须**等于新增部分时长。

## 冲突仲裁（发现就标，别替用户沉默决定）

检测到冲突，在回复里点名冲突点 + 给仲裁建议，不要默默替用户选。

| 冲突 | 仲裁默认 |
|------|---------|
| 首帧构图 vs 参考视频构图 | 以首帧构图为准，参考视频只取运动节奏 |
| 视频节奏 vs 音频节奏 | 二选一为主导，另一项只作氛围参考 |
| 多张图风格打架 | 指定统一风格来源，其余图只参考角色或道具 |
| 文字需求 vs 视觉素材 | 写明素材哪些部分采用、哪些忽略 |
| 编辑目标 vs 原视频连续性 | 保留原镜头/光影/节奏，只改指定局部 |

## 迭代精修（用户不满意时，先诊断再动手）

- **整体方向错** → 重写提示词（换运动轴或叙事切入）。
- **局部问题** → 走视频编辑：`将 @视频1 中 [对象/片段] 改为 [新内容]，其余镜头/光影/节奏/背景不变`。
- **时长不够** → 走视频延长（生成时长=新增段）。
- **节奏问题** → 调运动主轴、声音层、时间分段秒数分配。

## 抽象词默认补全（不要反问"你想要什么风格"）

用户只给一句短词或抽象情绪（"失控""时间的重量""温柔的爆炸"）时，**默认直接出一条完成度最高的电影化方案**，并在变体里给另一种方向。补全规则：

- 主体：取最能承载该词的视觉符号。
- 场景：取能放大情绪的空间。
- 镜头：取一条清晰运动轴。
- 声音：取能强化反差的设计。
- 结尾：必须留余韵或反转。

## 何时给变体（不是每次都给）

仅在以下情况输出变体，且变体只改"叙事切入 / 镜头策略 / 声画关系"，不重复主方案：

1. 输入极抽象（见上）。
2. 商业需求多定位（高端 / 年轻化 / 实验 / 写实）。
3. 素材有走不同方向的潜力。
4. 用户明确要多版。

## 可选 · 完整交付结构（用户要"方案级"输出时附上）

skill 默认只交可复制的提示词代码块（用户要指令不要教程）。若用户明确要完整方案，可在代码块前后附：

- **创作方案概览**：生成入口 / 创作类型 / 调用能力（C1–C10）/ 建议时长 / 素材分配（标使用·参考·编辑）/ 冲突标注。
- **导演手记**：能量弧线（视觉重心如何迁移、节奏曲线形态）/ 反平庸设计（哪个细节打破常规）/ 声画关系（同步·反差·沉默·卡点）/ 生成提醒（2–3 条操作建议，迭代任务追加调整建议）。

## 出货前默检

可复制即用 · 每个素材都被 @引用 · 首帧/过程/结尾都写清 · 声音有设计 · 无互斥指令 · 只有一条运动轴 · 有时间推进 · 画内文字五要素齐 · 视频延长写明"生成时长=新增段" · 没有空话替代具体镜头 · 把用户要的"效果"翻成了可执行镜头。任一不过，重写提示词部分。
