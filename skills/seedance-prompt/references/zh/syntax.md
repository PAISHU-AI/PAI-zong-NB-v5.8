# Seedance 2.0 — @语法 · 运镜 · 结构（中文）

> 按需加载模块。硬性上限与流程见 SKILL.md。

## 用 @ 指派素材

每个上传按顺序命名：图片是 `@图片1 … @图片9`，视频是 `@视频1 … @视频3`，音频是 `@音频1 … @音频3`。写 Seedance 提示词，核心几乎全在于——明确告诉模型每个素材**干什么用**。角色要说出口，模型不擅长从一句空泛的提及里猜。

| 你想让素材承担的角色 | 有效写法 |
|---|---|
| 作为开场首帧 | `以 @图片1 开场` |
| 作为收尾尾帧 | `最后一帧落在 @图片2` |
| 作为人物 | `主角用 @图片1 的人物` |
| 定场景 | `放在 @图片3 的环境里` |
| 决定运镜路径 | `沿用 @视频1 的运镜路径` |
| 决定动作 | `复刻 @视频1 里的身体动作` |
| 决定特效与转场 | `完全套用 @视频1 的转场和特效` |
| 决定节奏 | `按 @视频1 的节奏剪` |
| 决定音色 | `旁白音色照 @视频1` |
| 决定配乐 | `用 @音频1 配乐` |
| 决定音效 | `音效从 @视频2 里取` |
| 给人物换装 | `让主角穿 @图片2 的服装` |
| 锁定产品外观 | `产品忠于 @图片3` |
| 决定字幕样式 | `字体样式照 @图片2` |

需要时一行里叠几个：
```
主角用 @图片1 的人物，沿用 @视频1 的运镜路径和节奏，
用 @音频1 配乐，放在 @图片2 的环境里
```

## 运镜术语表

都是标准影视术语，原样写进去，模型会照做。

**基础运镜**——推近 / 拉远；左右摇；上下俯仰；跟拍（镜头随主体）；环绕（绕主体转）；固定机位（不动）；一镜到底（不切）。

**特殊运镜**——希区柯克变焦（推拉+反向变焦的"眩晕"感）；鱼眼；低角度仰拍（向上看，主体显得高大）；高角度 / 俯拍；第一人称主观；甩镜（带模糊的横向急甩）；升降臂（弧线升降）；机械臂多角度跟随。

**景别**——极致特写（一只眼、一只手）；面部特写；中近景（头肩）；中景（腰以上）；全景（全身）；远景 / 建立镜头（整个空间）。

## 结构展开

从主体往外搭：
```
[画面上是谁/什么]
[周围的环境]
[在做的动作，用朴素动词]
[镜头怎么动]
[超过8秒就分时间节点]
[任何切换或特效]
[配乐、音效、人声]
[整体观感与情绪]
```
超过 8 秒就切成节点，让模型自己控节奏，而不是赶：
```
0–3秒    建立场景，镜头开始运动
3–7秒    主要动作展开
7–11秒   转折或高潮
11–15秒  收束，最后一帧，字幕卡
```

## 容易翻车的地方

- 没说用途的引用（"用 @视频1"）——说清取哪一层：路径？动作？转场？节奏？
- 同一节点里相互打架的指令——0–3秒不能既"固定机位"又"环绕"。
- 塞太多——四秒只装一个清楚的动作，不是四个场景。
- 没人认领的素材——上传九张图，每张都要有明确用途。
- 没有声音方案——音频是生成的，要么引导它，要么接受默认。
- 复杂度超过时长——你要的量级要配得上你选的秒数。
- 上传真人可辨识脸——会被拒，改用生成的角色。
