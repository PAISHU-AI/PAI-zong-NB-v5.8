# pdp-cro-audit

## Purpose

Review a product detail page and produce a prioritized conversion improvement plan.

## Workflow

1. Inspect available page copy, screenshots, product facts, or URL content.
2. Check promise clarity, image coverage, price/offer clarity, proof, objections, CTA, FAQ, and mobile scanning.
3. Separate quick copy fixes from design, media, merchandising, or platform changes.
4. Rank issues by likely impact and effort.
5. Provide test ideas without claiming measured lift.

## Required Checks

- Do not invent page metrics.
- Do not recommend platform edits without confirmation.
- If no page evidence is available, return an audit checklist and request inputs.

## Output

Return findings, priority table, quick wins, deeper tests, and missing evidence.
